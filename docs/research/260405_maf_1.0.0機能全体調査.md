## 1. Agent (ChatAgent)

`Agent` は `ChatAgent` のエイリアス。LLM呼び出し・ツール使用・マルチターン会話・構造化出力・ストリーミングを統合。

### コンストラクタ (主要パラメータ)

```python
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient

agent = Agent(
    client=OpenAIChatClient(model="gpt-4o", azure_endpoint="...", api_key="..."),
    name="my-agent",
    instructions="あなたは親切なアシスタントです。",
    tools=[my_func],                    # FunctionTool, 関数, MCP ツール
    middleware=[my_middleware],          # Agent/Chat/Function ミドルウェア
    context_providers=[history_provider],
    temperature=0.7,
    max_tokens=4096,
    response_format=MyPydanticModel,    # 構造化出力
    tool_choice="auto",                 # "auto" | "required" | "none"
)
```

全パラメータ: `client`, `instructions`, `id`, `name`, `description`, `tools`, `middleware`, `context_providers`, `chat_message_store_factory`, `conversation_id`, `allow_multiple_tool_calls`, `model_id`, `temperature`, `max_tokens`, `top_p`, `frequency_penalty`, `presence_penalty`, `stop`, `seed`, `logit_bias`, `response_format`, `tool_choice`, `store`, `user`, `metadata`, `additional_chat_options`

### 実行

```python
# 非ストリーミング
result = await agent.run("質問")  # → AgentResponse
print(result.text)

# ストリーミング
async for chunk in agent.run("質問", stream=True):  # → ResponseStream
    print(chunk.text, end="")

# 最終結果も取得可能
stream = agent.run("質問", stream=True)
async for chunk in stream:
    print(chunk.text, end="")
final = await stream.get_final_response()
```

### Agent-as-Tool

```python
sub_agent = Agent(client=client, name="specialist", instructions="...")
tool = sub_agent.as_tool(name="SpecialistLookup", description="...", arg_name="query")
main_agent = Agent(client=client, tools=[tool])
```

## 2. Tools

6つの定義方法がある。

### 2.1 素の Python 関数

```python
def get_weather(location: str) -> str:
    """指定された場所の天気を取得する。"""
    return f"{location}は晴れです"

agent = Agent(client=client, tools=[get_weather])
```

### 2.2 @tool デコレータ

```python
from typing import Annotated
from agent_framework import tool

@tool(name="weather", description="天気を取得")
def get_weather(location: Annotated[str, "場所名"]) -> str:
    return f"{location}は晴れです"
```

### 2.3 FunctionTool

```python
from agent_framework import FunctionTool

tool = FunctionTool(
    func=my_func,
    name="custom_name",
    description="説明",
    input_model=MyPydanticModel,  # or JSON schema dict
    approval_mode="always_require",  # HIL 承認
    max_invocations=5,
)
```

### 2.4 Agent-as-Tool

前述の `agent.as_tool()` パターン。`propagate_session=True` でセッション共有可能。

### 2.5 MCP ツール

```python
from agent_framework import MCPStdioTool, MCPStreamableHTTPTool, MCPWebsocketTool

# stdio ベース
mcp_tool = MCPStdioTool(command="uvx", args=["my-mcp-server"])

# HTTP ストリーム

# WebSocket

```

### 2.6 ホステッドツール (OpenAI クライアント提供)

```python
from agent_framework.openai import OpenAIChatClient

client = OpenAIChatClient(model="gpt-4o")

```

## 3. Workflow

グラフベース実行エンジン。modified Pregel (BSP: Bulk Synchronous Parallel) モデル。

### 基本構築

```python
from agent_framework import WorkflowBuilder, Executor, handler

class Processor(Executor):
    @handler
    async def handle_input(self, ctx, message: str) -> None:
        result = process(message)
        ctx.send_message(Validator, result)

class Validator(Executor):
    @handler
    async def handle_result(self, ctx, message: str) -> None:
        if valid(message):
            ctx.yield_output(message)

workflow = (
    WorkflowBuilder(start_executor=Processor)
    .add_edge(Processor, Validator)
    .build()
)

result = await workflow.run("入力データ")
```

### Edge タイプ

| タイプ | メソッド | 用途 |
|--------|---------|------|
| Single | `add_edge(A, B)` | A→B 直列接続 |
| Chain | `add_chain([A, B, C])` | A→B→C 連鎖 |
| Fan-Out | `add_fan_out_edges(A, [B, C])` | A→B, A→C 並列分岐 |
| Fan-In | `add_fan_in_edges([B, C], D)` | B→D, C→D 合流 |
| Switch-Case | `add_switch_case_edges(A, [(cond1, B), (cond2, C)])` | 条件分岐 |

### チェックポイント

```python
from agent_framework import FileCheckpointStorage

storage = FileCheckpointStorage(directory="./checkpoints")
workflow = WorkflowBuilder(...).build(checkpoint_storage=storage)
```

### Human-in-the-Loop

```python
class MyExecutor(Executor):
    @handler
    async def handle(self, ctx, message):
        # ユーザーに情報要求 → ワークフロー一時停止
        response = await ctx.request_info("確認してください: ...")
```

## 5. Memory / State

### AgentSession

```python
session = agent.create_session()  # or AgentSession()
result = await agent.run("質問", session=session)

# 状態の読み書き
session.state["counter"] = 1
serialized = session.to_dict()
restored = AgentSession.from_dict(serialized)
```

### ContextProvider

`before_run` / `after_run` フックで実行コンテキストを動的に注入。

```python
from agent_framework import ContextProvider

class MyProvider(ContextProvider):
    def __init__(self):
        super().__init__(source_id="my-provider")

    async def before_run(self, agent, session, context, state):
        context.instructions += "\n追加の指示"
        context.tools.append(extra_tool)

    async def after_run(self, agent, session, context, state):
        # レスポンス処理
        pass
```

### HistoryProvider (会話履歴永続化)

```python
from agent_framework import InMemoryHistoryProvider

# インメモリ (デフォルト)
provider = InMemoryHistoryProvider()

# Cosmos DB 永続化
from agent_framework.azure import CosmosHistoryProvider
provider = CosmosHistoryProvider(
    endpoint="https://<account>.documents.azure.com:443/",
    credential=DefaultAzureCredential(),
    database_name="agent-framework",
    container_name="chat-history",
)

agent = Agent(client=client, context_providers=[provider])
```

ゼロコンフィグ: `AgentSession` を渡すだけで `InMemoryHistoryProvider` が自動追加される。

## 6. Compaction (メモリ圧縮)

長い会話でコンテキストウィンドウを超えないための戦略。

| 戦略 | 説明 |
|------|------|
| `SlidingWindowStrategy(keep_last_groups=N)` | 直近N組のメッセージのみ保持 |
| `TruncationStrategy` | トークン数で切り詰め |
| `SummarizationStrategy` | LLMで古い部分を要約 |
| `TokenBudgetComposedStrategy` | 複数戦略を組み合わせ |
| `SelectiveToolCallCompactionStrategy` | ツール呼び出し結果のみ圧縮 |
| `ToolResultCompactionStrategy` | ツール結果を圧縮 |

```python
from agent_framework import SlidingWindowStrategy

agent = Agent(
    client=client,
    compaction_strategy=SlidingWindowStrategy(keep_last_groups=10),
)
```

## 7. Middleware

3層のインターセプトポイント。onion/wrapper パターン。

| 層 | クラス | 介入ポイント |
|----|--------|-------------|
| Agent | `AgentMiddleware` | `agent.run()` 全体 |
| Chat | `ChatMiddleware` | LLM API 呼び出し |
| Function | `FunctionMiddleware` | ツール実行 |

### デコレータパターン

```python
from agent_framework import agent_middleware, function_middleware

@agent_middleware
async def logging_middleware(context, call_next):
    print(f"Agent run started: {context.agent.name}")
    result = await call_next(context)
    print(f"Agent run completed")
    return result

@function_middleware
async def approval_middleware(context, call_next):
    if context.tool.name == "dangerous_tool":
        # 承認ロジック
        pass
    return await call_next(context)

agent = Agent(client=client, middleware=[logging_middleware, approval_middleware])
```

## 8. Evaluation

### ビルトイン評価

```python
from agent_framework import evaluate_agent, evaluator, keyword_check, tool_called_check

@evaluator
def is_concise(response: str) -> bool:
    return len(response.split()) < 500

results = await evaluate_agent(
    agent=my_agent,
    queries=["質問1", "質問2"],
    evaluators=[is_concise, keyword_check("期待キーワード")],
    expected_outputs=["期待出力1", "期待出力2"],
    num_repetitions=3,
)
```

### ビルトインチェック関数

| 関数 | 用途 |
|------|------|
| `keyword_check(keyword)` | レスポンスにキーワードが含まれるか |
| `tool_called_check(tool_name)` | 指定ツールが呼ばれたか |
| `tool_calls_present()` | いずれかのツールが呼ばれたか |
| `tool_call_args_match(tool_name, args)` | ツール引数が一致するか |

### Workflow 評価

```python
from agent_framework import evaluate_workflow

results = await evaluate_workflow(
    workflow=my_workflow,
    queries=["入力1"],
    evaluators=[my_evaluator],
)
```

## 9. Skills (実験的)

## 次のステップ候補

1. **Tool 定義の検証** — `@tool` デコレータと `FunctionTool` で playground にツール付きエージェントを追加

5. **Evaluation 構築** — `evaluate_agent()` でエージェント品質の自動テスト
6. **Streaming 実装** — API レイヤーで `ResponseStream` を SSE に変換
7. **MCP 統合** — 外部 MCP サーバーをツールとして接続
